# 需要避开的结构

## 二元反转

这些结构制造戏剧感，却让文本像模板。直接说判断。

| 模式 | 问题 |
|------|------|
| “不是 X，而是 Y” | 预设反转，读者已经猜到后半句 |
| “X 不是问题，Y 才是” | 公式化重构 |
| “真正的答案不是 X，而是 Y” | 用“真正”制造权威 |
| “表面是 X，本质是 Y” | 常见内容号结构 |
| “与其说 X，不如说 Y” | 绕路表达 |
| “从 X 到 Y 的转变” | 虚假变化弧线 |
| “不只是 X，更是 Y” | 加法套话 |
| “X 看似简单，实则复杂” | 空泛反差 |

**改法：**直接说 Y。比如“问题是 Y”“团队卡在 Y”“Y 造成了这次延期”。

## 否定铺垫

先列一串“不是”，再揭示“是”，会显得像演讲稿。

| 模式 | 问题 |
|------|------|
| “不是 A，不是 B，而是 C” | 用否定堆气氛 |
| “它既不是 A，也不是 B，它是 C” | 同一套结构换说法 |
| “你缺的不是 A，也不是 B，而是 C” | 社媒训话腔 |

**改法：**直接说 C。读者不需要助跑。

## 三段式口号

中文 AI 文本喜欢把判断切成三项，制造整齐感。

| 模式 | 问题 |
|------|------|
| “认知、行动、复盘” | 抽象名词堆叠 |
| “看见问题，解决问题，成就自己” | 口号，不是信息 |
| “效率。质量。体验。” | 用碎片代替句子 |
| “先理解，再执行，最后沉淀” | 流程正确但没有内容 |

**改法：**删到一项或两项。保留能推进文本的信息。

## 设问自答

这些句子宣布洞察，而不是给出洞察。

| 模式 | 问题 |
|------|------|
| “为什么会这样？因为...” | 讲师口吻 |
| “这意味着什么？” | 多余过渡 |
| “你有没有想过...” | 假装对话 |
| “答案很简单...” | 贬低问题复杂度 |
| “那我们该怎么办？” | 模板化转段 |

**改法：**把答案写成陈述句。问题只在确实需要读者停下来判断时保留。

## 模板句

| 模式 | 问题 |
|------|------|
| “在 X 的背景下” | 常见论文腔开头 |
| “随着 X 不断发展” | 空泛时间推进 |
| “面对 X，我们需要 Y” | 政策稿口吻 |
| “只有 X，才能 Y” | 绝对化 |
| “这不仅是 X，更是 Y” | 机械升级 |
| “从 X 到 Y” | 假装结构完整 |
| “围绕 X 展开 Y” | 动作不清楚 |

**改法：**删掉框架词，写人、动作、结果。

## 虚假主体

不要让抽象名词替人行动。数据不会说话，市场不会亲自奖励你，趋势不会自己重塑业务。

| 模式 | 问题 |
|------|------|
| “问题暴露出...” | 问题没有暴露，某个人发现了 |
| “数据告诉我们...” | 数据摆在那里，人读数据 |
| “市场选择了...” | 买家选择，用户付费 |
| “组织开始意识到...” | 具体管理者或团队意识到 |
| “文化发生转变” | 人改变了行为 |
| “趋势正在重塑...” | 具体技术、政策或用户行为造成变化 |
| “需求催生了...” | 某群人提出需求，某个团队做了产品 |
| “价值被释放” | 谁拿到了什么收益 |

**改法：**写出人。比如“客服团队上周修了这个流程”胜过“问题推动流程优化”。

## 远距离旁白

作者飘在现场外面，句子就会发空。

| 模式 | 问题 |
|------|------|
| “很多人会...” | 没有具体人 |
| “人们往往...” | 旁观者口吻 |
| “这个现象说明...” | 课堂总结 |
| “这就是为什么...” | 多余解释 |
| “没人会...” | 懒惰绝对词 |

**改法：**把读者放进现场。写“你打开报表，只看到三个红色指标”，不要写“很多人面对数据会感到困惑”。

## 被动和无主句

被动句隐藏责任。无主句隐藏行动者。

| 模式 | 改法 |
|------|------|
| “方案被提出” | 写谁提出方案 |
| “问题得到解决” | 写谁解决问题 |
| “共识已经形成” | 写哪些人同意了什么 |
| “相关工作正在推进” | 写谁在推进哪件事 |
| “决定已经作出” | 写谁决定 |
| “错误被发现” | 写谁发现错误 |

**改法：**找到行动者，把行动者放到句首。

## 句子开头

这些开头容易把句子带进模板。

| 模式 | 改法 |
|------|------|
| “当我们谈论...” | 直接说讨论对象 |
| “如果说...” | 删掉假设壳 |
| “为什么...” | 改成陈述句 |
| “如何...” | 写具体动作 |
| “在 X 中” | 删掉背景框 |
| “从 X 角度来看” | 写判断，不写角度 |
| “对于 X 来说” | 把 X 放进主句 |
| “关于 X” | 直接进入 X |

不是所有这类开头都错。连续出现时，说明文本在绕路。

## 节奏模式

| 模式 | 改法 |
|------|------|
| 三项并列 | 用一项或两项 |
| 连续短句口号 | 合并成完整句 |
| 每段都用金句收尾 | 用事实、动作或问题收尾 |
| 破折号制造停顿 | 改成逗号、句号或重排 |
| 设问后马上回答 | 直接陈述 |
| “不完美，但足够好” | 写清楚哪里不完美，为什么能用 |

## 词语模式

| 模式 | 问题 |
|------|------|
| 懒惰绝对词，比如“所有”“任何”“永远”“从不”“没人”“大家都” | 假权威。改成范围、样本或具体对象 |
| 空泛程度词，比如“非常”“深度”“高度”“全面”“显著” | 没有尺度 |
| 名词堆叠，比如“用户增长路径优化能力建设” | 没有人、动作和结果 |
| 翻译腔，比如“进行一个评估”“做出一个决定” | 中文里有更短的动词 |

写具体句子，不堆概念。
